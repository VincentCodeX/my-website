# 從 AntiGravity 2.0 談起：我的部落格寫作自動化奇幻旅程

今天對我來說是充滿收穫的一天。我深入研究了 Google 最近推出的 **AntiGravity 2.0**，並在反覆的嘗試與除錯中，對「自動化工作流」有了全新的體會與靈感。

---

## 1. AntiGravity 2.0 的核心轉變：IDE 與 Codex 專案的分流

這次 AntiGravity 2.0 最大的動作，就是將功能拆分為兩個核心模組。這種細分讓我非常有感：

* **專門撰寫程式碼的 IDE 軟體**：將編寫環境單獨拉出來，讓開發者能更專注於程式碼本身的邏輯與語法。
* **Codex / Co-work 專案模式**：類似於近期非常熱門的 ChatGPT 與 Claude 協作生態。它可以讓我們設立一個專案，並在其中針對整個「工作自動化流程」進行架構與佈局。

### 我的體驗心得

以前開發常遇到「一邊要想整體流程，一邊還要分心寫細部代碼」的窘境，導致大腦容易打結、思緒中途斷掉。

透過這次的功能細分，我可以在 Codex 中像總指揮一樣先把自動化框架勾勒出來，再到 IDE 軟體裡專心把程式碼補齊，這讓思緒變得清晰許多。

---

## 2. 滾動式學習：今日的最大進步點

剛接觸 AntiGravity 2.0 的新架構時，確實花了不少時間摸索。在建置與整理的過程中，毫不意外地遇到了許多報錯（Errors）。

然而，這也是今天最讓我感到成就感的地方：

1. **發現問題**：遇到報錯不氣餒。
2. **截圖發問**：將錯誤畫面一一截圖詢問 AI。
3. **驗證與除錯**：根據 AI 的建議反覆測試、修正、再驗證。

透過這種反覆的實戰流程，我目前已經掌握了大概 **七到八成** 的操作。

這種滾動式的除錯過程，比看任何說明書都還要紮實，是我今天最大的進步點！

---

## 3. 未來的終極目標：部落格寫作自動化工作流

理清了 AntiGravity 2.0 的專案邏輯後，我也明確了自己最想實現的第一個自動化腳本——**「聊天對話 ➔ 一鍵生成 Markdown ➔ 自動部署網站」**。

### 🚀 理想中的自動化工作流（Workflow）

```text
【 步驟一：靈感激盪 】
在聊天視窗中自由對話、交流想法，理清零碎的思緒。
       │
       ▼
【 步驟二：腳本觸發（終點站） 】
話題告一段落時，一鍵觸發自動化腳本。
腳本將對話精華自動萃取，並生成帶有 Front Matter 的標準 .md 檔案。
       │
       ▼
【 步驟三：自動存檔與偵測 】
生成的 Markdown 檔案自動儲存至部落格的指定資料夾。
       │
       ▼
【 步驟四：後台自動部署 】
後台的 VS Code 偵測到新檔案，自動同步並上傳部署至個人網站。
```
